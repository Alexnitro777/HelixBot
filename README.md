# 🧬 HelixBot

> **HelixBot** — это современный, масштабируемый и полностью модульный Discord-бот на TypeScript, разработанный с акцентом на высокую отказоустойчивость, гибкую конфигурацию под каждый сервер и быструю интеграцию с существующими базами данных MariaDB/MySQL через Drizzle ORM.

---

## 📂 1. Структура проекта

Скелет проекта организован по принципу строгого разделения ответственности и независимости модулей:

```text
HelixBot/
├── .env.example            # Шаблон конфигурации переменных окружения
├── .gitignore              # Правила исключения файлов из контроля версий Git
├── Dockerfile              # Двухэтапная (multi-stage) сборка Docker-образа
├── docker-compose.yml      # Описание сервиса бота в режиме network_mode: host
├── drizzle.config.ts       # Конфигурация для Drizzle Kit (миграции)
├── package.json            # Зависимости и скрипты сборки
├── tsconfig.json           # Строгие настройки TypeScript (strict mode)
└── src/
    ├── index.ts            # Точка входа (миграции, инициализация, graceful shutdown)
    ├── config/
    │   └── config.ts       # Валидация конфигурации через Zod
    ├── core/
    │   ├── client.ts       # HelixClient (загрузчик модулей, кэширование, маршрутизатор событий)
    │   └── types.ts        # Описание типов и контрактов команд, модулей и слушателей
    ├── db/
    │   ├── index.ts        # Пул подключений mysql2 и инстанс Drizzle ORM
    │   └── schema.ts       # Описание реляционной схемы БД (guilds, guild_modules)
    ├── logger/
    │   └── index.ts        # Настройка быстрого логирования через Pino
    └── modules/            # Папка для функциональных модулей бота
        └── embed/          # Модуль отправки информационных сообщений (Embeds)
            ├── index.ts    # Инициализация модуля, Slash-команда и слушатели
            ├── registry.ts # Динамический загрузчик локальных шаблонов
            ├── types.ts    # Типовые спецификации шаблонов
            └── embeds/
                └── info.ts # Пример шаблона (информационное меню с кнопками)
```

## ⚡ 2. Быстрый старт с Docker

### 📋 2.1 Предварительные требования

Перед началом развертывания убедитесь, что на сервере подготовлены:
1. **Docker** и плагин **Docker Compose**.
2. Установленная локальная база данных **MariaDB** или **MySQL**.
3. **База данных и пользователь** (рекомендуется создать их через phpMyAdmin).
4. **Discord Bot Token** и **Client ID** (создаются на [Discord Developer Portal](https://discord.com/developers/applications)).

> [!IMPORTANT]
> Бот работает в режиме `network_mode: "host"`, что позволяет ему без дополнительной маршрутизации подключаться к СУБД, установленной непосредственно на хост-сервере по адресу `127.0.0.1`.

### 🚀 2.2 Установка и запуск

1. **Подготовьте окружение**  
   Создайте локальный файл конфигурации на основе примера:
   ```bash
   cp .env.example .env
   ```

2. **Настройте переменные окружения**  
   Откройте созданный файл `.env` и введите данные:

   | Переменная | Описание | Значение по умолчанию |
   | :--- | :--- | :--- |
   | `DISCORD_TOKEN` | Токен вашего Discord-бота | *Обязательно* |
   | `DISCORD_CLIENT_ID`| ID вашего Discord-приложения | *Обязательно* |
   | `DB_HOST` | Хост СУБД MariaDB / MySQL | `127.0.0.1` |
   | `DB_PORT` | Порт подключения к СУБД | `3306` |
   | `DB_USER` | Имя пользователя базы данных | `bot_user` |
   | `DB_PASSWORD` | Пароль пользователя базы данных | `bot_password` |
   | `DB_NAME` | Название базы данных | `bot_database` |
   | `LOG_LEVEL` | Уровень логирования pino | `info` |

   > [!NOTE]
   > Таблицы в базе данных создавать вручную не нужно. При первом запуске бот автоматически применит миграции Drizzle и создаст необходимые структуры.

3. **Сборка и запуск контейнера**  
   Запустите бота в фоновом режиме одной командой:
   ```bash
   docker compose up -d --build
   ```

## 🛠️ 3. Управление контейнером

Используйте стандартные команды Docker для контроля работы бота:

* **Просмотр логов в реальном времени**:
  ```bash
  docker logs -f helixbot
  ```

* **Перезапуск бота**:
  ```bash
  docker restart helixbot
  ```

* **Остановка бота и удаление контейнера**:
  ```bash
  docker compose down
  ```
